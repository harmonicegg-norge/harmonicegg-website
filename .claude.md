# Harmonic Egg Website - Claude Context

## Project Overview
Static website for Harmonic Egg Norge (harmonicegg.no) - a wellness/healing business in Oslo.
Built to replace expensive Squarespace site ($200/month → $40/month savings).

## Tech Stack
- **11ty (Eleventy)** - Static site generator
- **Nunjucks** - Templating engine
- **Vanilla JavaScript** - No frameworks (keep it simple!)
- **Custom CSS** - No frameworks, custom responsive design
- **Netlify** - Planned hosting platform

## File Structure
```
src/
├── _data/site.js          # ALL site data (prices, services, contact)
├── _includes/base.njk     # Main layout (header, footer, meta tags)
├── assets/
│   ├── css/style.css      # All styling
│   └── js/main.js         # All JavaScript
├── pages/                 # Individual pages
│   ├── booking.njk
│   ├── kontakt.njk
│   └── priser.njk
├── index.njk              # Homepage
└── images/                # Images (mostly placeholders currently)
```

## Important Files
- **src/_data/site.js** - Central data store (edit here for prices, services, etc.)
- **src/_includes/base.njk** - Base layout template (header, footer, scripts)
- **src/assets/css/style.css** - All styling (responsive, mobile-first)
- **src/assets/js/main.js** - Mobile menu, carousels, FAQ, animations
- **.eleventy.js** - 11ty configuration

## Common Commands
```bash
npm start              # Dev server → http://localhost:8080 (with live reload)
npm run build          # Build static files to _site/
npm run clean          # Remove _site/ folder
```

## Language & Content
- **Content Language:** Norwegian (bokmål)
- **Code/Comments:** English
- **Target Audience:** Norwegian wellness seekers, 30-65 years old
- **Tone:** Professional, calm, trustworthy, healing-focused

## Coding Conventions
- **Keep it simple** - Vanilla JS, no frameworks
- **Mobile-first** - Responsive design (breakpoint: 768px)
- **Static only** - No server-side code
- **Semantic HTML** - Good for SEO and accessibility
- **Progressive enhancement** - Works without JS
- **Data-driven** - Edit site.js, not templates

## What NOT to Do
- ❌ Don't add React/Vue/other heavy frameworks
- ❌ Don't overcomplicate JavaScript
- ❌ Don't add server-side rendering
- ❌ Don't add complex build processes
- ❌ Don't use jQuery or other libraries
- ❌ Keep dependencies minimal
